# Guia de Contribuição

Ficamos muito felizes pelo seu interesse em contribuir para este projeto! Toda a ajuda é bem-vinda. Este documento fornece um conjunto de diretrizes para contribuir com o projeto.

## Como Posso Contribuir?

### Reportar Bugs
-   Se encontrar um bug, por favor, certifique-se de que ele ainda não foi reportado, pesquisando na secção de [Issues](https://github.com/your-username/your-repository/issues) no GitHub.
-   Se não houver um relatório existente, por favor, [abra uma nova issue](https://github.com/your-username/your-repository/issues/new/choose), utilizando o template de **Bug Report**. Certifique-se de incluir um título claro e uma descrição detalhada, com os passos para reproduzir o problema.

### Sugerir Funcionalidades
-   Se você tem uma ideia para uma nova funcionalidade, sinta-se à vontade para [abrir uma nova issue](https://github.com/your-username/your-repository/issues/new/choose) utilizando o template de **Feature Request**.
-   Descreva a sua ideia de forma clara, explicando o problema que ela resolve e a solução que você propõe.

## Fluxo de Pull Request

1.  **Faça o Fork do Repositório:** Comece por fazer um [fork](https://github.com/your-username/your-repository/fork) deste repositório.
2.  **Crie uma Branch:** Crie uma nova branch a partir da `main` para as suas alterações. Use um nome descritivo (ex: `feature/add-billing-page` ou `fix/login-error`).
    ```bash
    git checkout -b nome-da-sua-branch
    ```
3.  **Faça as Suas Alterações:** Implemente o seu código ou faça as alterações na documentação.
4.  **Commit das Alterações:** Faça o commit das suas alterações com uma mensagem clara e concisa.
    ```bash
    git commit -m "feat: Adiciona a funcionalidade X"
    ```
5.  **Envie para o GitHub (Push):**
    ```bash
    git push origin nome-da-sua-branch
    ```
6.  **Abra um Pull Request:** Vá ao repositório no GitHub e abra um novo Pull Request. Preencha o template do PR com as informações solicitadas.
7.  **Revisão de Código:** Aguarde a revisão do seu PR. Os mantenedores do projeto poderão solicitar alterações.
8.  **Merge:** Uma vez aprovado, o seu PR será integrado (merged) à branch `main`.

Obrigado por contribuir!
