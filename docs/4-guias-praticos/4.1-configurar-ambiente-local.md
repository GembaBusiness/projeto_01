# 4.1 Guia: Configurar o Ambiente Local

Este guia descreve os passos necessários para configurar e executar o projeto na sua máquina local.

## Pré-requisitos

Antes de começar, certifique-se de que tem as seguintes ferramentas instaladas:
- [Git](https://git-scm.com/)
- [Node.js](https://nodejs.org/) (versão 18 ou superior)
- [Docker](https://www.docker.com/products/docker-desktop/) (para o Supabase local)
- [Supabase CLI](https://supabase.com/docs/guides/cli)

## Passos

### 1. Clonar o Repositório

Primeiro, clone o repositório do projeto do GitHub para a sua máquina local.

```bash
git clone <URL_DO_REPOSITORIO>
cd <NOME_DO_DIRETORIO_DO_PROJETO>
```

### 2. Instalar Dependências do Frontend

O nosso frontend é uma aplicação Node.js. Instale as dependências usando `npm` ou `yarn`.

```bash
# Navegue para a pasta do frontend (ex: 'app' ou 'frontend')
cd app

# Instale as dependências
npm install
```

### 3. Configurar o Supabase Local

Utilizamos a Supabase CLI para executar uma instância local do Supabase, que inclui a base de dados PostgreSQL, autenticação e APIs.

```bash
# A partir da raiz do projeto, inicie o Supabase
supabase start
```

Este comando irá iniciar vários contentores Docker. No final, a CLI irá mostrar-lhe informações importantes, incluindo:
- **API URL**: O URL base para a sua API local.
- **DB URL**: A string de conexão para a base de dados.
- **Studio URL**: Um link para o Supabase Studio, uma interface gráfica para gerir a sua base de dados local.
- **JWT Secret**: O segredo para assinar os tokens JWT.
- **Anon Key** e **Service Role Key**: As chaves de API para interagir com o seu Supabase local.

### 4. Aplicar as Migrações da Base de Dados

Para criar o esquema da base de dados na sua instância local, execute as migrações existentes.

```bash
supabase db reset
```

Este comando irá apagar os dados existentes e aplicar todos os ficheiros de migração da pasta `supabase/migrations`.

### 5. Configurar as Variáveis de Ambiente

A aplicação frontend precisa de saber como se conectar ao Supabase local. Crie um ficheiro `.env.local` na pasta do frontend (e.g., `app/.env.local`) e adicione as chaves fornecidas pelo comando `supabase start`.

```
NEXT_PUBLIC_SUPABASE_URL=http://127.0.0.1:54321
NEXT_PUBLIC_SUPABASE_ANON_KEY=<SUA_ANON_KEY_AQUI>
```

### 6. Executar a Aplicação

Agora que tudo está configurado, inicie o servidor de desenvolvimento do frontend.

```bash
# Na pasta do frontend
npm run dev
```

Abra o seu navegador em `http://localhost:3000` (ou a porta indicada no terminal) e deverá ver a aplicação a funcionar.
