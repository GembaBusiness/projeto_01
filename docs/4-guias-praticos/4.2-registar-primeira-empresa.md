# 4.2 Guia: Registar a Primeira Empresa

Este guia mostra o fluxo de chamadas de API necessárias para registar um novo utilizador e, em seguida, criar a sua primeira empresa.

## Fluxo Geral

O processo consiste em três passos principais:
1.  **Registar um novo utilizador**: O utilizador cria uma conta com email e password.
2.  **Fazer Login**: O utilizador troca as suas credenciais por um `access_token`.
3.  **Criar a Empresa**: Com o `access_token`, o utilizador chama a função `create_company_and_profile` para criar a empresa e ser associado a ela.

---

### Passo 1: Registar o Utilizador (Sign Up)

Esta chamada cria uma nova entrada em `auth.users` e, graças ao nosso `trigger`, também cria um perfil correspondente em `public.profiles`.

**Requisição:**
`POST /auth/v1/signup`

**Cabeçalhos:**
- `apikey`: `SUPABASE_ANON_KEY`

**Corpo (JSON):**
```json
{
  "email": "novo.utilizador@example.com",
  "password": "uma_password_muito_segura"
}
```

**Resposta de Sucesso (200 OK):**
A resposta incluirá os detalhes do novo utilizador, mas ainda sem um `access_token`.

---

### Passo 2: Fazer Login (Obter Token)

Agora, o utilizador troca as credenciais que acabou de criar por um token de sessão.

**Requisição:**
`POST /auth/v1/token?grant_type=password`

**Cabeçalhos:**
- `apikey`: `SUPABASE_ANON_KEY`

**Corpo (JSON):**
```json
{
  "email": "novo.utilizador@example.com",
  "password": "uma_password_muito_segura"
}
```

**Resposta de Sucesso (200 OK):**
A resposta conterá o `access_token` e o `refresh_token`. Guarde o `access_token` para o próximo passo.

```json
{
  "access_token": "eyJhbGciOiJIUzI1Ni...",
  "token_type": "bearer",
  ...
}
```

---

### Passo 3: Criar a Empresa

Com o token de acesso, o utilizador pode agora chamar a função RPC para criar a empresa.

**Requisição:**
`POST /rest/v1/rpc/create_company_and_profile`

**Cabeçalhos:**
- `apikey`: `SUPABASE_ANON_KEY`
- `Authorization`: `Bearer <O_ACCESS_TOKEN_DO_PASSO_2>`

**Corpo (JSON):**
```json
{
  "company_name": "A Minha Primeira Empresa"
}
```

**Resposta de Sucesso (200 OK):**
A resposta será o `id` da empresa recém-criada.

```json
"c1b2a3d4-e5f6-7890-1234-567890abcdef"
```

## Verificação

Após estes passos, pode verificar na sua base de dados que:
- Existe um novo utilizador em `auth.users`.
- Existe um novo perfil em `public.profiles`.
- Existe uma nova empresa em `public.companies`.
- Existe um novo vínculo em `public.memberships` que liga o utilizador à empresa com o papel de `admin`.
