## 1. Visão Geral da Arquitetura
Esta página fornece uma visão geral dos componentes que formam o nosso sistema, utilizando o C4 Model como referência para os diagramas.

### 1.1. Diagrama de Contexto (Nível 1)
O Diagrama de Contexto mostra o sistema como uma "caixa preta" e como ele interage com utilizadores e outros sistemas externos.



#### 1.1.1. Descrição dos Atores e Sistemas
- **Utilizador Final (Administrador da Empresa):** Interage com o sistema através da aplicação web (SPA) para gerir a sua empresa, utilizadores e outras funcionalidades.
- **Sistema Externo de Pagamentos:** Processa transações financeiras e gestão de subscrições. (ex: Stripe)
- **Serviço de Email Transacional:** Envia notificações, convites e emails relacionados a eventos no sistema. (ex: Postmark, SendGrid)

### 1.2. Diagrama de Contentores (Nível 2)
O Diagrama de Contentores foca nos principais blocos de construção (aplicações e armazenamentos de dados) que compõem o sistema.

#### 1.2.1. Componentes Principais
- **Single Page Application (SPA):** A nossa aplicação frontend construída com WeWeb, que gera código Vue.js. É responsável por toda a interface e experiência do utilizador.
- **Backend as a Service (BaaS) - Supabase:** O coração do sistema, que providencia um conjunto de serviços integrados sobre uma base de dados PostgreSQL.
  - **Base de Dados (PostgreSQL):** Armazena todos os dados da aplicação, utilizando Row-Level Security (RLS) para garantir o isolamento de dados entre tenants.
  - **Autenticação:** Gere o ciclo de vida dos utilizadores (registo, login, recuperação de senha).
  - **Edge Functions (Deno):** Funções serverless que orquestram a lógica de negócio mais complexa e segura.
  - **Armazenamento de Ficheiros:** Para armazenar ficheiros e documentos dos utilizadores (ex: logos de empresas, avatares).

#### NOTA: A escolha pelo Supabase permite-nos acelerar o desenvolvimento, delegando a gestão da infraestrutura de backend e focando na lógica de negócio, que é implementada de forma segura através da combinação de políticas de RLS e Edge Functions.
