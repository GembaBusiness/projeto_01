# 1.1 Visão Geral da Arquitetura

Esta página fornece uma visão geral dos componentes que formam o nosso sistema, utilizando o C4 Model como referência para os diagramas.

## Diagrama de Contexto (Nível 1)

O Diagrama de Contexto mostra o sistema como uma "caixa preta" e como ele interage com utilizadores e outros sistemas.

_Aqui iria um diagrama gerado ou uma imagem, por exemplo:_
`![Diagrama de Contexto](caminho/para/diagrama-contexto.png)`

**Descrição:**
- **Utilizador Final**: Interage com o sistema através da nossa aplicação web (SPA).
- **Sistema Externo de Pagamentos**: Processa transações financeiras.
- **Serviço de Email**: Envia notificações e emails transacionais.

## Diagrama de Contentores (Nível 2)

O Diagrama de Contentores foca nos principais blocos de construção (aplicações e armazenamentos de dados) dentro do sistema.

_Aqui iria um diagrama, por exemplo:_
`![Diagrama de Contentores](caminho/para/diagrama-contentores.png)`

**Componentes Principais:**
- **Single Page Application (SPA)**: A nossa aplicação frontend construída em [React/Vue/Angular]. É responsável pela interface do utilizador.
- **API Backend**: O coração do sistema, construído em [Node.js/Python/Go]. Expõe os endpoints para a SPA e lida com a lógica de negócio.
- **Base de Dados**: Uma base de dados PostgreSQL gerida pelo [Supabase/AWS RDS] que armazena todos os dados da aplicação.
- **Serviço de Ficheiros**: [Supabase Storage/AWS S3] para armazenar ficheiros e documentos dos utilizadores.
